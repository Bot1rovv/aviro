# Arivoo – Маркетплейс товаров из Китая

Проект представляет собой интернет-магазин (маркетплейс) для поиска и покупки товаров с китайских площадок Taobao, 1688 и Poizon. Реализован на
Next.js 16 с использованием App Router, TypeScript, Tailwind CSS и Zustand для управления состоянием. Интегрирован с реальным API Daji для получения
актуальных данных о товарах.

## 🚀 Функциональность

- **Поиск товаров** по ключевым словам с фильтрацией по источникам (Taobao, 1688, Poizon) и пагинацией
- **Детальные страницы товаров** с галереей изображений, видео, переведённым описанием, спецификациями, вариантами (SKU) и информацией о доставке
- **Аутентификация** – регистрация, вход, выход, защищённый профиль пользователя (с использованием httpOnly cookie, JWT, bcrypt)
- **Корзина** – добавление/удаление товаров, изменение количества, оформление заказа в два шага
- **Избранное** – сохранение товаров для последующего просмотра с синхронизацией через localStorage
- **Профиль пользователя** – вкладки «Заказы», «История действий», «Настройки»
- **Адаптивный дизайн** – мобильная и десктопная версии с отдельными layout-компонентами
- **Модальные окна** – корзина, условия использования, меню на мобильных устройствах, согласие на cookie
- **Страница 404** – кастомная страница для несуществующих маршрутов
- **Соглашение на использование cookie** – баннер с запросом согласия, сохранение предпочтений
- **Категории товаров** – иерархическая навигация по категориям 1688 с автоматическим переводом на русский
- **Кэширование API** – in‑memory LRU кэширование для снижения нагрузки на внешние API и ускорения повторных запросов
- **Rate limiting** – защита API аутентификации от брут‑форса
- **Toasts (уведомления)** – использование библиотеки Sonner для информирования пользователя

## 🔐 Аутентификация

Реализована с помощью:

1. **API маршрутов** (`/api/auth/login`, `/api/auth/register`, `/api/auth/logout`)
2. **HTTP‑only cookie** – токен сохраняется в защищённом cookie (`auth_token`), что повышает безопасность
3. **JWT с секретом из переменных окружения** – подпись и верификация через библиотеку `jose`
4. **Хеширование паролей bcrypt** – пароли никогда не хранятся в открытом виде
5. **Rate limiting** – ограничение попыток входа/регистрации
6. **Mock-логика** (для демонстрации, можно заменить на реальную БД)
7. **Хранилище пользователя** (`lib/store/user.ts`) с синхронизацией через cookie

После входа в шапке вместо иконки замка отображается имя пользователя и ссылка на профиль.

## 🍪 Управление cookie

- **Согласие на cookie** – компонент `CookieConsent` запрашивает разрешение на использование файлов cookie при первом посещения.
- **Хранилище предпочтений** – Zustand-хранилище `cookie-consent` сохраняет выбор пользователя в localStorage.
- **Безопасные cookie** – для аутентификации используются httpOnly, secure, sameSite cookie.

## ⚡ Кэширование API

Для снижения нагрузки на внешние API и ускорения повторных запросов реализовано in‑memory LRU кэширование:

- **Модуль кэша** – `lib/utils/cache.ts` предоставляет кэш на основе LRU с настраиваемым TTL и размером.
- **TTL и размер** – по умолчанию TTL 5 минут, максимальный размер 100 записей в памяти.
- **Интеграция с API товаров** – маршрут `/api/product/[id]` проверяет кэш перед запросом к внешнему сервису.
- **Ключи кэша** – формируются по параметрам запроса (id товара, источник).
- **Сброс кэша** – при изменении данных можно очистить кэш через вызов `clearCache()`.

## 🔗 Интеграция с внешними API

Проект использует **Daji API** (`https://openapi.dajisaas.com`) для доступа к товарам Taobao, 1688 и Poizon.

- **Подпись запросов** – каждый запрос подписывается MD5-хешем на основе секретного ключа (реализация в `lib/signature.ts`).
- **Обогащение параметров** – автоматическое добавление `appKey`, `appSecret`, `language`, `sign` и других обязательных полей.
- **Обработка ошибок** – корректная обработка сетевых ошибок и возвратов от API.
- **Поддержка трёх источников** – единый интерфейс `product-service` для поиска и получения деталей товаров.

## 🛒 Корзина и избранное

- Состояние корзины и избранного хранится в Zustand-сторах
- Данные сохраняются в localStorage для persistence
- Модальное окно корзины с возможностью быстрого редактирования
- Пошаговое оформление заказа (выбор товаров → ввод данных → подтверждение)

## 🔧 Переменные окружения

Создайте файл `.env.local` в корне проекта и определите:

```env
# Порт (опционально)
PORT=3001

# Секреты для JWT и API Daji
JWT_SECRET=your_super_secret_jwt_key_change_in_production
NEXT_PUBLIC_DAJI_ACCESS_KEY=your_daji_access_key
NEXT_PUBLIC_DAJI_ACCESS_SECRET=your_daji_access_secret

# Базовый URL API (обычно оставляем по умолчанию)
NEXT_PUBLIC_API_BASE_URL=https://openapi.dajisaas.com

# Параметры кэша (опционально)
CACHE_MAX_SIZE=100
CACHE_TTL_MS=300000
```

**Внимание**: В демонстрационных целях в коде уже указаны тестовые ключи, но для production необходимо использовать свои.

## 🔄 Рефакторинг и улучшения кода

На основе анализа кодовой базы выявлены возможности для улучшения архитектуры и устранения дублирования:

1. **Вынесение общих констант** – цвета источников (`sourceColors`) вынесены в `lib/constants/source-colors.ts` и используются через компонент
   `SourceBadge`.
2. **Кастомные хуки** – планируется создание `useCartActions`, `useFavoritesActions` для инкапсуляции логики работы с корзиной и избранным.
3. **Унификация градиентов** – часто используемый градиент `from-[#0d65df] to-[#0752c2]` будет вынесен в Tailwind-конфиг как кастомный класс.
4. **Использование существующих UI-компонентов** – замена ручной разметки на `EmptyState`, `SourceBadge`, `PriceDisplay` и др.
5. **Применение утилит форматирования** – использование `formatPrice` из `lib/utils/format.ts` для единообразного отображения цен.
6. **Рефакторинг модальных окон** – унификация логики открытия/закрытия через хук `useModal`.
7. **Типизация** – все типы экспортируются из `types/`, устранение inline-типов.
8. **Обработка ошибок API** – централизованный error handler и хук `useApi`.

Эти улучшения повысят поддерживаемость, уменьшат количество кода и обеспечат единообразие.
