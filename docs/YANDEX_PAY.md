## Обзор

Яндекс Пэй — платёжный сервис для онлайн и офлайн-магазинов. Поддерживает:
- Карта Пэй
- Банковские карты
- СБП (Система Быстрых Платежей)
- Оплата частями (Сплит)

**Официальная документация:** https://pay.yandex.ru/docs/ru/

---

## Начало работы

### 1. Подача заявки
Зарегистрируйтесь в [личном кабинете](https://pay.yandex.ru/docs/ru/console/registration) и подайте заявку на подключение.

### 2. Тестовая среда
После одобрения вы получите доступ к тестовой среде для отладки интеграции.

### 3. Переход на боевую среду
После успешного тестирования:
- Отключите "Тестовые данные" в настройках
- Заполните **Callback URL**
- Выпустите API-ключ
- Проведите финальное тестирование

## Виды платежей

### Одностадийные платежи
Средства списываются сразу после авторизации.

```
Создание (PENDING) → Оплата → Оплачен (CAPTURED)
```

### Двухстадийные платежи
Средства блокируются, затем подтверждаются продавцом.

```
Создание (PENDING) → Авторизован (AUTHORIZED) → Подтверждение → Оплачен (CAPTURED)
```

> ⚠️ Двухстадийные платежи требуют отдельного подключения через поддержку.

## Способы оплаты

| Способ | Тип | Доступен по умолчанию |
|--------|-----|----------------------|
| Карта Пэй | Способ | ✅ |
| Банковская карта | Способ | ✅ |
| СБП | Способ | ✅ (одностадийный) |

### Виды оплаты
- **Полная оплата** — 100% стоимости + кешбэк баллами Плюса
- **Оплата частями (Сплит)** — оплата по графику, продавец получает полную сумму сразу

## Интеграция

### План интеграции

1. **Подать заявку** → дождаться активации тестовой среды
2. **Выбрать платежную механику**
3. **Установить SDK** (Web, Android или iOS)
4. **Тестирование в тестовой среде**
5. **Переход на боевую среду**
6. **Настроить фискализацию**

### Типы интеграции

- **Для онлайн-магазинов** — Web SDK, Android SDK, iOS SDK
- **Для офлайн-магазинов** — QR-код от Яндекс Пэй
- **Через CMS модули** — готовые модули для популярных платформ

### Требования

> ⚠️ Запрещено встраивать Яндекс Пэй в WebView при подключении в мобильных приложениях — это небезопасно.

## API Endpoints

### Основные endpoints

| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | `/order/create` | Создание платежа |
| POST | `/order/{orderId}/capture` | Подтверждение двухстадийного платежа |
| POST | `/order/{orderId}/cancel` | Отмена платежа |
| POST | `/order/{orderId}/refund` | Возврат платежа |
| GET | `/order/{orderId}` | Получение информации о заказе |

### Создание платежа (order/create)

**Request:**

```json
{
  "amount": {
    "value": "100.00",
    "currency": "RUB"
  },
  "payment_method": {
    "type": "CARD",
    "save_payment_method": false
  },
  "confirmation": {
    "type": "REDIRECT",
    "return_url": "https://example.com/success"
  },
  "capture": true,
  "description": "Заказ №123",
  "metadata": {
    "order_id": "123"
  }
}
```

**Response:**

```json
{
  "id": "order_123",
  "status": "PENDING",
  "amount": {
    "value": "100.00",
    "currency": "RUB"
  },
  "confirmation": {
    "type": "REDIRECT",
    "confirmation_url": "https://payment.yandex.ru/pay/..."
  }
}
```

### Подтверждение платежа (capture)

Используется для двухстадийных платежей.

```json
POST /order/{orderId}/capture

{
  "amount": {
    "value": "100.00",
    "currency": "RUB"
  }
}
```

### Отмена платежа (cancel)

```json
POST /order/{orderId}/cancel

{
  "reason": "Отмена заказа покупателем"
}
```

### Возврат платежа (refund)

```json
POST /order/{orderId}/refund

{
  "amount": {
    "value": "50.00",
    "currency": "RUB"
  }
}
```

## Статусы платежей

| Статус | Описание |
|--------|----------|
| `PENDING` | Создание платежа |
| `AUTHORIZED` | Платеж авторизован (двухстадийный) |
| `CAPTURED` | Оплачен |
| `CANCELED` | Отменён |
| `REFUNDED` | Возвращён |

## Операции с платежами

### Создание платежа
Через личный кабинет или API (`POST /order/create`)

### Возврат
- **Полный** — возврат всей суммы
- **Частичный** — возврат части суммы

### Отмена
Используется для отмены авторизованного платежа до его подтверждения.

### Прерывание платежа (rollback)
Автоматическая отмена при истечении срока блокировки (для двухстадийных).

## Webhook (Callback)

Для получения уведомлений о статусах платежей настройте **Callback URL**.

### Пример webhook payload

```json
{
  "type": "PAYMENT_NOTIFICATION",
  "event": "payment.capture",
  "object": {
    "id": "order_123",
    "status": "CAPTURED",
    "amount": {
      "value": "100.00",
      "currency": "RUB"
    }
  }
}
```

## Пример интеграции (Web SDK)

### 1. Подключение скрипта
```html
<script src="https://payment.yandex.net/yap.js"></script>
```

### 2. Инициализация кнопки
```javascript
const payment = new YaPay({
  env: 'sandbox', // или 'production'
  merchantId: 'YOUR_MERCHANT_ID',
  sessionTimeout: 300
});

payment.on('pay-success', (data) => {
  console.log('Оплата успешна:', data);
});

payment.on('pay-error', (data) => {
  console.log('Ошибка оплаты:', data);
});
```

### 3. Создание кнопки
```html
<div id="yapay-button"></div>
```

## Тестирование

### Тестовая среда
- Включите "Тестовые данные" в настройках личного кабинета
- Испольйте тестовые карты:
  - Успешная оплата: `5555 5555 5555 4444` (любой срок, любой CVC)
  - Отклонённая оплата: `5555 5555 5555 0000`

### Тестирование в боевой среде
После перехода на production проведите тестовые платежи с реальными картами.
